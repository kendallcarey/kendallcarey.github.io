Welcome back to my technical blog. I missed you too. The topic chosen today (did you pick this?) is the Enumerable Cycle.

I know you’re thinking, why would I use the enumerable cycle? Is it useful? What is an enumerable? Enumerable is a fun word to say. Indubitably. Well, I am here to answer all your questions… Well, at least about the enumerable cycle in Ruby anyway. I’m not going to cover the meaning of life in this short tech blog... 42.

The enumerable module offers a set of methods search, sort and manipulate arrays and hashes. It simplifies. It makes your code “prettier,” at least most of the time.

Now, the enumerable method “cycle” does pretty much what you would expect it to do, it cycles through your array or hash however many times you like or even infinitely. It’s a bit cleaner than using .times. You can use the enumerable cycle (the enumerable cycle is starting to sound like a super hero’s name) with the enumerable #next to retrieve the next element as many times a necessary - which I would like to point out that you CANNOT do with the enumerable#each (unlike each, it returns a fresh enumerator everytime!). Although you can use each and cycle together which is useful if you want to reverse cycle through your collection as there is no reverse cycle enumerable but there is a reverse each enumerable.

Well, I hope this was useful and if it wasn’t, I hope you do your research and find an enumerable that will be beneficial for your needs. Until next time!

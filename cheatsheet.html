Cheat Sheet for Ruby Enumerables By: Kendall and Bridgette

DEFINITION

Enumerables are a set of methods to traverse, search, sort and manipulate collections.

all?

Passes each element of the collection to the given block and returns true or false if the block never returns false.

any?

Similar to all but it passes each element of the collection to the given block and returns true if the block returns ever returns a value other than false.

chunk

Takes an array and separates the consecutive elements that have the same block value.

collect

Outputs a new array that contains the results of a given block after its been applied once for each element.

count

Can be used like .length but if an argument is given, it will count the number of items in the collection.

cycle

Calls block on enum repeatedly n times. If n is not given, this process will repeat forever.

detect

Given a block, it goes through the collection and returns the first for which block is true and nil if all are false.

drop

You insert a number in the argument and drop will delete that many elements in the collection.

each_cons

Iterates a block n times for each consecutive element in an array

each_slice

Devides n sections, or n "slices", of an array into groups and returns them in new arrays.

each_with_index

Calls a block with 2 arguments and returns each element in the collection with its corresponding index number.

each_with_object

Iterates the given block for every element, requiring an arbitrary object to be given, then returns the initial object reflection changes.

find

Exactly like detect, in fact, the example used in ruby docs is exactly the same as the the example for detect

find_all

Runs a given block then returns a new array containing elements for which the block evaluated to true.

find_index

Compares all elements in enum to a given block and returns the index of the first element that meets the conditions of the given block.

first

Returns the first element in a collection.

grep

Very popularly used with Regex, it returns an array of every element in the collection for which the pattern===element. For example: (1..100).grep 38..44 #=> [38, 39, 40, 41, 42, 43, 44]

group_by

Returns a hash where keys = result from a given block and the values are arrays of elements in enum that correspond to the key. This groups enum by the result of the block given.

include?

Returns true if any element in the collection equals the object.

inject

Can be supplemented with reduce. If using inject, give a block. The block will iterate through the collection and return the result.

map

Will run the block once for every element in enum and then return an array with the results

max

Returns the object in the collection that has the greatest value.

max_by

Pretty much the same as 'max' but calls a block. Even Ruby-docs gives the same definition.

member?

Returns true if any member of enum equals argument passed in

min

Returns the object in enum with the smallest value

min_by

Same as 'min' but calls a block.

minmax

Returns two elements in an array that contains the min and max values in enum

minmax_by

Same as minmax but calls a block.

none?

Passes each element from a collection through a block and returns true if the block never returns true for any element.

one?

Runs each element through a block and returns true if conditions of the block are met once. If no block is given, this will return true if ONLY one of the collection members evaluates to true

partition

Will output two arrays. The first array will contain the elements of enum where the block evaluates to true and the second array will contain the false elements.

reduce

Combines all the elements in a collection and by applying an operation specified by a block or symbol that calls a method or operator.

reject

Will output an array for all members of a collection of enum for which a given block returns false.

reverse_each

creates a temporary array from enum and reverses the order of its elements

select

Outputs an array that contains the members of a collection that pass the conditions set in a given block.

sort

Returns an array containing the elements sorted either alphabetically or by using the results of the supplied block.

sort_by

Pretty much the same, just uses the results of the supplied block.

take

Returns the first n amount of elements from the collection.

take_by

Passes the elements through the block and returns the elements until the block returns false then stops and returns the elements that were true in an array.

to_a

Returns the numbers in the collection to an array.

to_h

Returns the list of keys and values in a hash.

zip

This one can be confusing, but it can take several arrays as an argument and merge them one element at a time into arrays of each index.
